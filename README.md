## Getting Started

You'll need to provide some configuration to connect to the Merged OpenAPI Specification - see the articles about [short-lived access tokens](https://support.lusid.com/knowledgebase/article/KA-01654) and a [long-lived Personal Access Token](https://support.lusid.com/knowledgebase/article/KA-01774). This configuration can be provided using a secrets file or environment variables.

For some configuration it is also possible to override the global configuration at the ApiFactory level, or at the request level. For the set of configuration which can be overridden, please see [ConfigurationOptions](sdk/Lusid/Extensions/ConfigurationOptions.cs). For a code illustration of this configuration being overridden, please see the [example](#example).

### Environment variables

Required for a short-lived access token
``` 
FBN_TOKEN_URL
FBN_BASE_URL
FBN_USERNAME
FBN_PASSWORD
FBN_CLIENT_ID
FBN_CLIENT_SECRET
```

Required for a long-lived access token
``` 
FBN_BASE_URL
FBN_ACCESS_TOKEN
```

You can send your requests to the Merged OpenAPI Specification via a proxy, by setting `FBN_PROXY_ADDRESS`. If your proxy has basic auth enabled, you must also set `FBN_PROXY_USERNAME` and `FBN_PROXY_PASSWORD`.

Other optional configuration

```bash
# the sdk client timeout in milliseconds, the default is 1800000 (30 minutes)
# values must be between 1 and 2147483647
# please note - the chances of seeing a network issue increases with the duration of the request
# for this reason, rather than increasing the timeout, it will be more reliable to use an alternate polling style endpoint where these exist
FBN_TIMEOUT_MS
# the retries when being rate limited, the default is 2
FBN_RATE_LIMIT_RETRIES
```

### Secrets file

The secrets file must be in the current working directory. By default the SDK looks for a secrets file called `secrets.json`

Required for a short-lived access token
```json
{
  "profiles": {
    "default": {
        "tokenUrl":"<your-token-url>",
        "baseUrl":"https://<your-domain>.lusid.com",
        "username":"<your-username>",
        "password":"<your-password>",
        "clientId":"<your-client-id>",
        "clientSecret":"<your-client-secret>"
    }
}
```

Minimum requirements for authentication with any finbourne API endpoint.
```json
{
  "profiles": {
    "default": {
        "baseUrl":"https://<your-domain>.lusid.com",
        "accessToken":"<your-access-token>"
    }
}
```

The Finbourne SDK also supports multiple profiles in the secrets file, which can be used to store different credentials for different environments (e.g. production, dev, qa). To specify which profile to use, set the `FBN_PROFILE` environment variable to the name of the relevant profile. 
You can also specify the profile to use at the ApiFactory level, or at the request level. For a code illustration of this configuration being overridden, please see the [example](#example).
```json
{
  "profiles": {
    "default": {
        "tokenUrl":"<your-token-url>",
        "baseUrl":"https://<your-domain>.lusid.com",
        "username":"<your-username>",
        "password":"<your-password>",
        "clientId":"<your-client-id>",
        "clientSecret":"<your-client-secret>"
    },
    "dev": {
        "tokenUrl":"<your-token-url>",
        "baseUrl":"https://<your-dev-domain>.lusid.com",
        "username":"<your-username>",
        "password":"<your-password>",
        "clientId":"<your-client-id>",
        "clientSecret":"<your-client-secret>"
    },
    "qa": {
        "tokenUrl":"<your-token-url>",
        "baseUrl":"https://<your-qa-domain>.lusid.com",
        "username":"<your-username>",
        "password":"<your-password>",
        "clientId":"<your-client-id>",
        "clientSecret":"<your-client-secret>"
    }
}  
```

### Authentication Example

The minimum requirements for authentication is having a `baseUrl` and an `accessToken`
The below will if one of the following assumptions are true:
- The `FBN_ACCESS_TOKEN` and `FBN_BASE_URL` environment variables are set
- A `secrets.json` file is present in the current working directory with the `baseUrl` and `accessToken` fields populated or contains user credentials that can be used to generate an access token
```csharp
    var apiFactory = new ApiFactoryBuilder(environmentVariables).Build();
    var lusidInstance  = apiFactory.Api<Lusid.Api.ApplicationMetadataApi>();
    
   lusidInstance.GetLusidVersions();
```


```csharp
    var environmentVariables = new Dictionary<string, string>
    {
        { "FBN_ACCESS_TOKEN", Environment.GetEnvironmentVariable("FBN_ACCESS_TOKEN") },
        { "FBN_BASE_URL", Environment.GetEnvironmentVariable("FBN_BASE_URL") }
    };
    // It is not a requirement to pass the environment variables the ApiFactoryBuilder, as it will automatically read from the environment variables if they are not passed in.
    var apiFactory = new ApiFactoryBuilder(environmentVariables).Build();
    var lusidInstance  = apiFactory.Api<Lusid.Api.ApplicationMetadataApi>();
    
   lusidInstance.GetLusidVersions();
```

Authenication with a secrets file
```csharp
    string mySecretsFile = "/path/to/my/secrets.json";
    var apiFactory = new ApiFactoryBuilder().Build(mySecretsFile);
    var lusidInstance  = apiFactory.Api<Lusid.Api.ApplicationMetadataApi>();

    lusidInstance.GetLusidVersions();
```

Selecting a profile from a secrets file, when selecting a profile the name must match what is in your secrets file.
```csharp
    string mySecretsFile = "/path/to/my/secrets.json";

    var config = new Dictionary<string, string>
    {
        { "FBN_PROFILE", "dev" }
    };

    var apiFactory = new ApiFactoryBuilder(config).Build(mySecretsFile);
    var lusidInstance  = apiFactory.Api<Lusid.Api.ApplicationMetadataApi>();

    lusidInstance.GetLusidVersions();
```
